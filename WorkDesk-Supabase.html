⬡ WorkDesk Supabase Setup — one-time configuration

Follow these steps once. Your settings are saved in your browser.

1

Run this SQL in Supabase → SQL Editor

Creates all tables WorkDesk needs. Safe to re-run.

-- WorkDesk schema (safe to re-run) create table if not exists workdesk_tasks ( id text primary key, user_id uuid references auth.users not null, data jsonb not null, updated_at timestamptz default now() ); create table if not exists workdesk_meta ( key text primary key, user_id uuid references auth.users not null, value jsonb not null, updated_at timestamptz default now() ); create table if not exists workdesk_sessions ( user_id uuid primary key references auth.users, session_token text not null, last_seen timestamptz default now() ); alter table workdesk_tasks enable row level security; alter table workdesk_meta enable row level security; alter table workdesk_sessions enable row level security; drop policy if exists "own tasks" on workdesk_tasks; drop policy if exists "own meta" on workdesk_meta; drop policy if exists "own session" on workdesk_sessions; create policy "own tasks" on workdesk_tasks for all using (auth.uid()=user_id); create policy "own meta" on workdesk_meta for all using (auth.uid()=user_id); create policy "own session" on workdesk_sessions for all using (auth.uid()=user_id);
2

Create your login — Supabase → Authentication → Users

Click Add user → Create new user. Set your email and password.

3

Paste your Supabase credentials

Found under Settings → API in your Supabase project.
